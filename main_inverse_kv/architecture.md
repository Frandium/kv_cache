# From Attention to Search: Dynamic Hierarchical Indexing for Million-Token Transformer Inference

## 1. Introduction

Large language models are rapidly entering an era of extremely long-context inference. Modern systems already support context windows extending into hundreds of thousands or even millions of tokens. While this scaling substantially expands the amount of information a model can access during inference, it also exposes a fundamental computational problem inside the Transformer architecture itself.

At inference time, when a new token is generated, the model constructs a query vector and compares it against all historical key vectors stored in the KV cache. If the context contains one million tokens, then every newly generated token must effectively interact with one million historical entries. The computational structure therefore resembles a linear scan through memory.

Mathematically, standard attention computes:

$$
a_i =
\operatorname{softmax}
\left(
\frac{q^\top k_i}{\sqrt d}
\right)
$$

where:

- \(q\) is the query vector of the current token,
- \(k_i\) is the key vector associated with historical token \(i\),
- \(d\) is the hidden dimension,
- and \(a_i\) is the normalized attention weight.

The computational complexity scales approximately linearly with context length:

$$
O(N)
$$

where \(N\) denotes the number of cached tokens.

At relatively small context lengths, this linear scan is manageable because modern GPUs are highly optimized for dense matrix multiplication. However, at one million tokens, the situation changes fundamentally. The bottleneck is no longer merely floating-point throughput. Instead, the system becomes dominated by memory movement, cache traversal, and increasingly sparse useful computation.

The crucial observation is that although attention mathematically evaluates every historical token, after softmax normalization only a tiny subset meaningfully contributes to the final output.

The softmax function is exponentially selective:

$$
\frac{a_i}{a_j} =
\exp(s_i - s_j)
$$

where:

$$
s_i =
\frac{q^\top k_i}{\sqrt d}
$$

Even a moderate logit difference creates an enormous disparity in final attention weights. If one token score exceeds another by 10, then its weight becomes approximately:

$$
\exp(10)
\approx
22026
$$

times larger.

This means that the overwhelming majority of tokens effectively vanish after normalization. Although one million dot products are computed, only a tiny fraction materially affects the output.

This observation fundamentally changes the nature of the problem.

Long-context attention is not truly a dense aggregation problem. It is more accurately an extreme-value retrieval problem. The real computational objective is not to interact uniformly with all tokens, but instead to identify the very small subset of tokens whose dot products with the query vector dominate after softmax normalization.

The question therefore becomes:

Can we avoid scanning the entire KV cache by constructing a searchable memory system capable of directly retrieving likely high-scoring candidates?

This naturally leads to a comparison with classical search systems such as Google Search. Search engines solved the problem of retrieving relevant information from enormous corpora decades ago. They achieved this not by linearly scanning the internet, but by constructing sophisticated indexing structures capable of reducing online search into highly localized retrieval operations.

This document explores the idea that long-context Transformer inference should increasingly be viewed not as a brute-force attention problem, but as a memory indexing and search problem.

# 2. Reinterpreting Transformer Attention as Search

The conceptual bridge between attention and search becomes clear once the components are interpreted from an information retrieval perspective.

In a search engine, a user submits a query, the system retrieves candidate documents from an indexed corpus, ranks the candidates according to relevance, and returns the highest-ranked results.

Transformer attention exhibits an almost identical structure.

The query vector generated by the current token behaves like a search query. Historical key vectors behave like indexed documents. Value vectors behave like payloads retrieved from memory. Attention logits behave like ranking scores.

The attention mechanism therefore resembles:

$$
q
\rightarrow
\{k_1,k_2,\dots,k_N\}
\rightarrow
\text{ranking}
\rightarrow
\text{weighted retrieval}
$$

This interpretation becomes even stronger when considering the selectivity of softmax normalization.

Traditional information retrieval systems often prioritize broad semantic recall. A search engine may prefer to retrieve many potentially relevant documents because multiple documents may collectively contribute useful information.

Transformer attention behaves differently. Due to exponential normalization, only the highest-scoring entries matter. Attention therefore behaves less like broad semantic retrieval and more like winner selection under extreme competition.

The true computational objective is closer to:

$$
\arg\max_i q^\top k_i
$$

rather than uniform interaction with all historical entries.

This distinction is critical because it changes the optimal systems design.

If the purpose of attention is to identify the extreme winners among one million candidates, then brute-force dense scanning becomes increasingly inefficient. The system should instead focus on rapidly locating the small subset of memory entries likely to contain the highest dot-product matches.

At this point, Transformer inference begins to resemble nearest-neighbor search over a dynamic vector database.

However, unlike traditional search systems, Transformer memory possesses several unusual properties:

1. The memory is generated dynamically during inference.
2. Every layer and attention head has its own representation space.
3. Memory insertion occurs continuously.
4. Retrieval latency must remain extremely low.
5. The objective is extreme-value precision rather than broad semantic recall.

These properties make the problem significantly different from classical document retrieval.

# 3. How Classical Search Engines Solved Scale

Before proposing a new architecture for Transformer memory systems, it is useful to examine how large-scale search systems historically solved retrieval at internet scale.

## 3.1 Inverted Indexing

The foundational innovation behind modern search systems is the inverted index.

A naive search engine might represent information as:

$$
\text{document}
\rightarrow
\{\text{terms inside document}\}
$$

Such a representation requires scanning every document during search.

The inverted index reverses the mapping:

$$
\text{term}
\rightarrow
\{\text{documents containing term}\}
$$

This transformation fundamentally changes the computational structure of retrieval. Instead of searching through all documents, the engine directly jumps to candidate lists associated with query terms.

The search process therefore becomes:

$$
\text{query terms}
\rightarrow
\text{posting lists}
\rightarrow
\text{candidate documents}
\rightarrow
\text{ranking}
$$

This principle is enormously important because it converts retrieval from exhaustive comparison into indexed lookup.

The central idea is not merely storage optimization. It is computational localization. The system narrows the search space before expensive ranking occurs.

## 3.2 Hierarchical Search Structures

As data scale increased, search systems evolved beyond simple inverted lists into increasingly hierarchical indexing structures.

Hierarchical clustering organizes documents into nested partitions. Instead of searching globally, the query first selects a coarse region of the search space, then progressively descends into finer partitions.

The structure resembles:

$$
\text{root cluster}
\rightarrow
\text{sub-cluster}
\rightarrow
\text{sub-sub-cluster}
\rightarrow
\text{candidate entries}
$$

Modern vector search systems employ similar ideas through methods such as:

- inverted file indexing (IVF),
- locality-sensitive hashing,
- product quantization,
- and graph-based nearest-neighbor search.

Although the implementations differ, the principle remains consistent:

Efficient retrieval emerges by partitioning the search space into structured regions.

## 3.3 Candidate Generation and Re-ranking

Search engines rarely perform expensive ranking over the entire corpus.

Instead, they decompose retrieval into two stages:

$$
\text{coarse retrieval}
\rightarrow
\text{fine ranking}
$$

The first stage rapidly generates a manageable candidate set. The second stage applies more expensive ranking algorithms only to those candidates.

This decomposition is deeply relevant to Transformer inference.

The expensive operation in long-context attention is not the final softmax over a small set of candidates. The expensive operation is computing interactions against the entire KV cache.

Therefore, an analogous architecture naturally emerges:

$$
\text{candidate retrieval}
\rightarrow
\text{exact sparse attention}
$$

## 3.4 Distributed and Incremental Indexing

Search systems also solved the problem of continuously evolving data.

Webpages change constantly. New pages appear continuously. Search engines therefore developed incremental indexing systems capable of dynamically updating searchable structures without rebuilding the entire index from scratch.

This idea becomes critically important for Transformer inference because the KV cache itself is continuously generated during decoding.

Unlike webpages, KV memory is not pre-existing. It is produced token by token during autoregressive inference.

This transforms the indexing problem from static retrieval into dynamic streaming memory maintenance.

# 4. Why Transformer KV Memory is Fundamentally Different

Although search systems provide important inspiration, Transformer memory differs fundamentally from classical search corpora.

## 4.1 Dynamically Generated Memory

Search engines operate primarily on relatively stable corpora.

Transformer memory behaves differently:

$$
\text{generate token}
\rightarrow
\text{generate KV}
\rightarrow
\text{insert into memory}
$$

Every generated token immediately becomes part of future searchable memory.

This means the indexing system must support continuous low-latency insertion during inference.

The KV cache therefore behaves more like a streaming vector database than a static document collection.

## 4.2 Layer-wise and Head-wise Representation Spaces

Transformer memory is not globally homogeneous.

Each layer and each attention head possesses its own representation geometry:

$$
\mathcal I =
\{
\mathcal I_{\ell,h}
\}_{\ell,h}
$$

where:

- \(\ell\) denotes the layer,
- \(h\) denotes the attention head.

The meaning of proximity inside one layer may differ completely from another layer.

Early layers may encode local syntax and positional continuity. Intermediate layers may encode phrase-level structures or entities. Late layers may encode semantic abstraction or reasoning dependencies.

Therefore, there is no single universal memory index. Each layer and head may require its own indexing structure.

## 4.3 Softmax Selectivity

Classical search systems often emphasize recall.

Transformer attention instead emphasizes extreme-value precision.

Because softmax is exponentially selective, most retrieved entries are useless unless they are near the maximum logit.

The true objective becomes:

$$
\max_i q^\top k_i
$$

rather than broad semantic retrieval.

This changes the design objective of the index itself. The goal is not retrieving many semantically related candidates. The goal is rapidly identifying the few tokens likely to dominate after softmax normalization.

## 4.4 Sequential and Temporal Structure

Transformer memory is sequentially generated.

Neighboring tokens are strongly correlated. Recency often matters. Positional locality remains important even in long-context reasoning.

Therefore, the memory system must combine semantic retrieval with temporal structure.

This differs from web search, where documents are largely independent.

## 4.5 Hardware Constraints

Transformer inference occurs under strict latency constraints.

Index insertion, routing, retrieval, and candidate expansion must all remain GPU-efficient.

An indexing system theoretically reducing asymptotic complexity but introducing irregular memory access patterns or excessive synchronization overhead may ultimately perform worse in practice.

Thus, the problem is simultaneously algorithmic, architectural, and hardware-oriented.

# 5. Proposed Architecture: MoE-Router-Aligned Dynamic Hierarchical KV Indexing

The previous sections argued that long-context Transformer inference should increasingly be viewed as a memory retrieval problem rather than merely a dense attention computation problem. We further argued that brute-force linear scanning of one million KV entries becomes increasingly inefficient because softmax attention is highly selective. Only a very small subset of tokens ultimately dominates the final output.

The earlier formulation proposed a dynamic hierarchical inverse index over the KV cache. However, an important question remained unresolved:

How should the memory bins themselves be defined?

A naive approach would introduce an external clustering algorithm or approximate nearest-neighbor index that periodically groups KV vectors into hierarchical regions. While such methods may improve retrieval efficiency, they introduce a major conceptual limitation. The indexing structure becomes disconnected from the Transformer’s own internal semantic organization.

Modern sparse Transformer architectures, particularly Mixture-of-Experts (MoE) models, already provide a potentially much more elegant solution.

The central insight of this revised architecture is that MoE routing itself already performs a learned partitioning of representation space.

Therefore, the MoE router can simultaneously serve two purposes:

1. sparse computational routing,
2. and sparse memory indexing.

This transforms the role of MoE fundamentally. Instead of merely reducing feed-forward compute cost, MoE routing becomes a semantic addressing system for long-context memory retrieval.

---

## 5.1 MoE Routing as Semantic Space Partitioning

In a standard MoE Transformer layer, the model first computes token representations after self-attention. A gating network then decides which expert or experts should process each token.

Conceptually, the router performs:

$$
x_i^\ell
\rightarrow
\operatorname{TopKExperts}(x_i^\ell)
$$

where:

- \(x_i^\ell\) denotes token \(i\) at layer \(\ell\),
- and the routing function selects one or several experts.

Suppose a layer contains 1,000 experts. Then the router effectively partitions the representation manifold into approximately 1,000 semantic regions.

Tokens routed to the same expert are implicitly judged by the model to share related computational characteristics. In other words, the expert assignment already behaves like a learned clustering label.

This observation is extremely important.

Traditional search systems require explicitly engineered indexing structures. However, MoE Transformers already contain an internal routing mechanism trained jointly with the model itself.

The router therefore becomes a learned semantic partitioning system emerging naturally from optimization.

Conceptually:

$$
\text{MoE routing}
\approx
\text{learned semantic clustering}
$$

This immediately suggests a new interpretation of KV cache organization.

When a token becomes part of the KV cache, its expert assignment can simultaneously serve as an indexing label.

Thus, instead of inserting a token into an externally constructed ANN structure, we can reuse the model’s own routing information.

---

## 5.2 Expert IDs as Dynamic Inverted Indexes

Consider a token position \(i\) processed at layer \(\ell\).

Suppose the router assigns the token to expert \(e\). Then, when the corresponding key-value pair enters the KV cache, we also insert the token position into the corresponding expert-index bucket.

The memory index therefore becomes:

$$
\mathcal I_{\ell,e} =
\{
i :
\text{token } i
\text{ routed to expert } e
\text{ at layer } \ell
\}
$$

This structure behaves like an inverted index.

In classical search systems:

$$
\text{term}
\rightarrow
\text{document IDs}
$$

In the proposed Transformer memory system:

$$
\text{expert ID}
\rightarrow
\text{token positions}
$$

This creates a powerful conceptual bridge between MoE routing and search indexing.

The important insight is that expert IDs are not arbitrary identifiers. They are learned semantic routing regions optimized jointly with the model itself.

Therefore, the indexing structure becomes model-native rather than externally imposed.

---

## 5.3 Query-Time Retrieval Through Router Alignment

During future inference steps, a new query token is processed by the same routing mechanism.

The new token representation is routed through the MoE gate:

$$
q_t^\ell
\rightarrow
\operatorname{TopKExperts}(q_t^\ell)
$$

The resulting expert assignments immediately determine which memory buckets should be searched.

The retrieval pipeline therefore becomes:

$$
q_t
\rightarrow
\text{router}
\rightarrow
\text{expert bins}
\rightarrow
\text{candidate KV entries}
\rightarrow
\text{exact sparse attention}
$$

This architecture is elegant because the same routing mechanism simultaneously governs both:

1. sparse feed-forward execution,
2. and sparse memory retrieval.

The MoE router therefore becomes a unified computational and memory-addressing mechanism.

This dual interpretation may represent one of the most important conceptual consequences of sparse MoE architectures.

---

## 5.4 Hierarchical Memory Organization

The expert assignment alone may still be too coarse.

Therefore, the KV memory hierarchy can be refined further.

The hierarchy may naturally contain several levels:

$$
(\ell,h,e,b)
\rightarrow
\text{token positions}
$$

where:

- \(\ell\) denotes layer,
- \(h\) denotes attention head,
- \(e\) denotes expert ID,
- and \(b\) denotes a finer local bin inside the expert region.

The expert ID performs coarse semantic partitioning.

The local bin performs finer-grained vector partitioning.

This creates a hierarchical memory organization:

$$
\text{layer}
\rightarrow
\text{head}
\rightarrow
\text{expert}
\rightarrow
\text{local semantic bin}
\rightarrow
\text{token positions}
$$

The resulting structure resembles a hybrid between:

- inverted indexing,
- hierarchical clustering,
- and learned semantic routing.

Importantly, the hierarchy is no longer externally engineered from scratch. It is partially induced by the Transformer itself.

---

## 5.5 Relationship to Softmax Selectivity

The MoE interpretation becomes even more compelling when combined with the earlier observation regarding softmax selectivity.

Attention does not require broad semantic recall.

Instead, the objective is identifying the very small number of tokens likely to dominate after exponential normalization.

The retrieval objective therefore becomes:

$$
\arg\max_i q_t^\top k_i
$$

rather than broad semantic retrieval.

If MoE experts already partition the semantic space into highly selective computational regions, then expert assignments naturally become good candidate generators for future softmax winners.

The router effectively predicts which semantic regions are likely relevant before exact attention occurs.

This transforms the problem fundamentally.

The system no longer performs brute-force comparison against one million tokens.

Instead, it first performs semantic routing, then sparse candidate retrieval, then exact local ranking.

Conceptually:

$$
\text{semantic routing}
\rightarrow
\text{candidate memory retrieval}
\rightarrow
\text{exact attention reranking}
$$

This architecture increasingly resembles modern search engines and memory hierarchies rather than traditional dense neural computation.

---

## 5.6 Dynamic Streaming Memory Maintenance

One of the most important aspects of this architecture is that the memory system evolves dynamically during autoregressive decoding.

Every generated token immediately produces:

1. new KV entries,
2. new routing assignments,
3. and new memory index insertions.

Thus, the inference loop becomes:

$$
\text{generate token}
\rightarrow
\text{compute routing}
\rightarrow
\text{insert KV into expert-indexed memory}
\rightarrow
\text{future retrieval}
$$

This makes the Transformer behave increasingly like a continuously evolving streaming database.

The KV cache is no longer merely a passive tensor buffer.

It becomes an actively maintained searchable memory hierarchy.

---

## 5.7 Computational Perspective

Traditional dense attention scales approximately as:

$$
O(N)
$$

with respect to context length.

The proposed architecture instead attempts to decompose the problem into:

$$
\text{coarse semantic routing}
+
\text{localized sparse retrieval}
+
\text{exact local attention}
$$

The idealized complexity becomes closer to:

$$
O(\log N)
+
O(m)
$$

where:

- \(m\) denotes the retrieved candidate size,
- and \(m \ll N\).

The exact scaling depends heavily on routing quality, cluster balance, insertion overhead, and hardware implementation.

However, the key conceptual transition remains clear:

The system no longer scans memory globally.

It navigates memory semantically.

# 6. Open Problems and Future Research Directions

Although the proposed architecture is conceptually appealing, many fundamental questions remain unresolved. These questions are not merely engineering details. They touch the deeper relationship between representation learning, memory organization, sparse computation, and retrieval geometry.

---

## 6.1 Do MoE Experts Truly Correspond to Semantic Regions?

The proposed framework assumes that experts behave like semantic partitions of representation space.

However, this assumption requires careful investigation.

Do experts genuinely specialize into coherent semantic regions?

Or do they instead behave as optimization artifacts reflecting load balancing and training dynamics?

If expert assignments are unstable or weakly semantic, then their usefulness as retrieval indexes may be limited.

Understanding the geometry and semantics of expert specialization therefore becomes a central research problem.

---

## 6.2 Alignment Between Routing and Attention Geometry

Another major question concerns the relationship between MoE routing and attention similarity.

The proposal assumes that tokens routed to the same expert are also likely to produce large attention logits:

$$
q^\top k
$$

However, routing and attention are optimized through different objectives.

The router determines feed-forward execution paths.

Attention determines memory interaction.

These objectives may not perfectly align.

Therefore, one major question is whether expert assignments actually correlate with future softmax winners.

If not, training objectives may need to explicitly encourage alignment between:

- routing geometry,
- and retrieval geometry.

---

## 6.3 Dynamic Cluster Balance

Large-scale indexing systems require balanced partitions.

If certain experts accumulate disproportionately many tokens, retrieval degenerates toward linear scanning inside overloaded buckets.

This creates an interesting connection between MoE load balancing and memory retrieval efficiency.

Current MoE systems already employ auxiliary balancing losses during training.

However, future architectures may require balancing not merely for compute utilization, but also for memory indexing quality.

Thus, expert balancing may become simultaneously:

- a computational scheduling problem,
- and a memory systems problem.

---

## 6.4 Temporal and Sequential Structure

Transformer memory is not an unordered database.

Tokens possess strong temporal correlations.

Recent tokens are often disproportionately important.

Neighboring tokens frequently share related semantic structure.

This raises important questions regarding hybrid retrieval strategies.

Should retrieval combine:

- semantic routing,
- temporal locality,
- recency bias,
- and structural position?

Future systems may require memory hierarchies resembling operating-system cache hierarchies:

- immediate local memory,
- medium-range indexed memory,
- compressed long-range semantic memory.

---

## 6.5 Hardware-Efficient Retrieval

Another major challenge concerns hardware implementation.

Modern GPUs are optimized for dense tensor computation with regular memory access patterns.

Hierarchical retrieval introduces irregularity.

Dynamic routing, sparse retrieval, and candidate expansion may introduce synchronization overheads, cache misses, and memory fragmentation.

Thus, a theoretically superior indexing algorithm may still perform poorly on real hardware.

This creates a critical co-design problem between:

- Transformer architecture,
- memory indexing structure,
- and accelerator hardware.

Future accelerators may require specialized support for sparse semantic memory retrieval.

---

## 6.6 Training-Time Co-Design

Current Transformer models are trained assuming dense attention.

The proposed architecture fundamentally changes the inference procedure.

Therefore, future models may require training objectives explicitly encouraging:

- routing stability,
- retrieval locality,
- semantic partition quality,
- and sparse memory accessibility.

This suggests an important shift:

The model may eventually learn not only how to compute, but also how to organize and retrieve its own memory efficiently.

---

## 6.7 Theoretical Questions

The proposal also raises deeper theoretical questions.

Why do only a few tokens dominate softmax competition?

What geometric structures emerge inside large-scale key spaces?

Can sparse semantic routing approximate dense attention without significant information loss?

Can memory indexing fundamentally alter long-context scaling laws?

Can retrieval-aware architectures achieve substantially larger effective context lengths under fixed computational budgets?

These questions suggest that future long-context Transformer research may increasingly intersect with:

- database systems,
- search engines,
- distributed memory architectures,
- operating systems,
- and information retrieval theory.

---

# Closing Perspective

The broader implication of this proposal is profound.

Mixture-of-Experts models were originally introduced primarily as sparse computation mechanisms.

However, this document suggests a second and potentially equally important interpretation:

MoE routing may also represent an emergent semantic memory addressing system.

If this perspective is correct, then future Transformer architectures may increasingly blur the boundary between:

- neural computation,
- semantic retrieval,
- hierarchical memory systems,
- and search engine architectures.

At sufficiently large context scales, the Transformer may cease to resemble a purely dense neural network.

Instead, it may increasingly resemble a dynamically evolving semantic operating system for memory retrieval and reasoning.
